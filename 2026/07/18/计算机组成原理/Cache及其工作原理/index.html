Cache 工作原理

Cache，即高速缓冲存储器，是位于 CPU 和主存之间的一种高速存储器。

它的主要作用是：

缓解 CPU 运算速度与主存访问速度之间的差距。

1. 为什么需要 Cache

CPU 的运行速度非常快，而主存 DRAM 的访问速度相对较慢。

如果 CPU 每次读取数据都直接访问 DRAM，就可能等待很长时间。因此，在 CPU 与主存之间增加了速度更快的 Cache。

计算机中的存储层次通常如下：

CPU 读取数据时，会先查找 L1 Cache。

如果 L1 中没有，再查找 L2；L2 中没有，再查找 L3；L3 中仍然没有，最后才访问主存 DRAM。

2. Cache 保存的是什么

Cache 保存的不是另一套独立的数据，而是：

但是，Cache 和主存之间通常不会只传输地址 1000 对应的一个字节，而是会把附近连续的一整块数据一起读入 Cache。

这个交换单位通常称为：

  • Cache Line
  • Cache Block
  • 高速缓存行
  • 高速缓存块

现代处理器常见的 Cache Line 大小是 64 字节，但具体大小由处理器决定。

主存与 Cache 之间按块交换数据，是为了利用程序的空间局部性。

3. 程序的局部性原理

Cache 能够提高性能，主要依赖程序访问数据时表现出来的局部性。

3.1 时间局部性

如果一个数据刚刚被访问过，那么它在短时间内可能再次被访问。

例如：

变量 sum 和 x 会被反复访问。

第一次读取这些变量后，它们被放入 Cache，后面的访问就可能直接从 Cache 中完成。

3.2 空间局部性

如果一个内存地址被访问，那么它附近的地址也可能很快被访问。

例如：

数组元素在内存中通常是连续存放的。

CPU 第一次访问 array[0] 时，会把包含 array[0] 的一整条 Cache Line 读入 Cache。

由于这条 Cache Line 中还包含后续的一些数组元素，因此访问 array[1]、array[2] 时，很可能直接命中 Cache。

5. Cache 中的数据是怎样组织的

Cache 由多条 Cache Line 组成。

一条 Cache Line 除了保存实际数据外，通常还包含一些管理信息：

Valid：有效位

Valid 用于表示当前 Cache Line 中的数据是否有效。

计算机刚启动或 Cache 刚初始化时，Cache Line 的 Valid 通常为 0。

Tag：标签

Tag 用来说明：

当前 Cache Line 中的数据来自主存的哪个区域。

不同的主存块可能映射到 Cache 中的同一个位置，因此必须通过 Tag 判断当前保存的数据是不是 CPU 真正需要的数据。

Dirty：脏位

Dirty 用于表示 Cache Line 中的数据是否被 CPU 修改过。

Dirty 位主要用于写回法。

Data：数据区

Data 是 Cache Line 中真正保存的数据，也就是从主存或下一级 Cache 复制过来的数据块。

12. Cache 满了怎么办

Cache 的容量远小于主存，因此 Cache 最终会被装满。

当发生 Cache Miss，需要装入新数据，但目标位置没有空闲 Cache Line 时，就必须选择一条旧的 Cache Line 替换掉。

这个过程称为：

Cache Replacement，Cache 替换。

常见的替换算法包括：

LRU

LRU 的全称是 Least Recently Used，即最近最少使用。

它会优先替换最长时间没有被访问的 Cache Line。

FIFO

FIFO 的全称是 First In First Out，即先进先出。

它会优先替换最早进入 Cache 的数据。

Random

随机选择一条 Cache Line 进行替换。

随机算法的硬件实现比较简单，在部分处理器中也会使用。

近似 LRU

完整实现 LRU 可能需要较多硬件资源，因此实际处理器经常使用近似 LRU 算法。

14. Cache 的三种映射方式

主存中的数据块放到 Cache 时，需要遵循一定的映射规则。

常见的映射方式有：

  1. 直接映射；
  2. 全相联映射；
  3. 组相联映射。

15. 直接映射

直接映射规定：

每个主存块只能放到 Cache 中唯一确定的位置。

通常可以通过下面的方式计算：

主存块 0 和主存块 8 都只能放到 Cache 行 0。

直接映射的优点是：

  • 硬件结构简单；
  • 查找速度快；
  • 成本较低。

缺点是：

  • 不同主存块可能频繁映射到同一条 Cache Line；
  • 容易产生冲突缺失。

例如，程序不断交替访问主存块 0 和主存块 8：

即使 Cache 中还有其他空闲位置，这两个主存块也不能放过去。

16. 全相联映射

全相联映射规定：

一个主存块可以放到 Cache 中的任意一条 Cache Line。

这种方式非常灵活，可以减少冲突缺失。

但是 CPU 查找数据时，需要把目标 Tag 与 Cache 中所有 Cache Line 的 Tag 同时比较。

优点是：

  • 数据块可以放到任意位置；
  • 冲突缺失较少；
  • Cache 空间利用率较高。

缺点是：

  • 需要大量并行比较电路；
  • 硬件复杂；
  • 成本较高；
  • Cache 越大，查找难度越高。

因此，全相联方式通常不适合直接用于容量较大的 Cache。

17. 组相联映射

组相联映射是直接映射和全相联映射的折中。

Cache 被划分为多个组，每组包含若干条 Cache Line。

18. CPU 写数据时如何处理

CPU 不仅会从 Cache 中读取数据，也会修改数据。

CPU 修改 Cache 中的数据后，需要处理 Cache 与下一层存储器之间的一致性问题。

常见写策略包括：

  1. 写直达；
  2. 写回。

19. 写直达 Write Through

写直达规定：

CPU 修改 Cache 的同时，也立即修改下一层存储器。

流程如下：

优点是：

  • Cache 与下一层存储器中的数据基本保持一致；
  • 控制逻辑相对简单；
  • Cache Line 被替换时通常不需要先写回。

缺点是：

  • 每次写操作都需要访问下一层存储器；
  • 写操作延迟较大；
  • 会增加存储总线压力。

为了减少 CPU 等待时间，写直达通常会配合写缓冲区使用。

20. 写回 Write Back

写回规定：

CPU 写数据时只修改 Cache，不立即更新下一层存储器。

修改后，将该 Cache Line 的 Dirty 设置为 1。

当这条 Cache Line 将来需要被替换时，再把整条 Cache Line 写回下一层存储器。

优点是：

  • 多次写同一条 Cache Line 时，不必每次访问主存；
  • 可以减少下一层存储器的写入次数；
  • 能够降低总线压力；
  • 写操作通常更快。

缺点是：

  • 控制逻辑更加复杂；
  • Cache 中的数据可能比主存中的数据新；
  • 替换脏 Cache Line 时需要额外写回操作。

21. 写缺失时如何处理

CPU 写入某个地址时，如果对应数据不在 Cache 中，就发生写缺失。

写缺失通常有两种处理方式。

写分配 Write Allocate

发生写缺失时，先将目标地址所在的数据块加载到 Cache，再在 Cache 中执行写操作。

写分配通常与写回策略配合使用。

非写分配 No Write Allocate

发生写缺失时，不将数据块装入 Cache，而是直接写入下一层存储器。

非写分配通常与写直达策略配合使用。

22. L1、L2、L3 Cache 如何配合

现代 CPU 通常具有多级 Cache：

一次读操作通常按照下面的顺序执行：

L1 Cache

L1 Cache 的特点通常是：

  • 容量最小；
  • 速度最快；
  • 距离 CPU 执行核心最近；
  • 一般由单个 CPU 核心独享；
  • 经常分成指令 Cache 和数据 Cache。

指令 Cache 用于保存 CPU 即将执行的机器指令。

数据 Cache 用于保存程序使用的数据。

L2 Cache

L2 Cache 通常：

  • 比 L1 容量大；
  • 比 L1 慢；
  • 一般仍由单个 CPU 核心独享；
  • 用于承接 L1 Cache Miss。

L3 Cache

L3 Cache 通常：

  • 比 L2 容量更大；
  • 比 L2 更慢；
  • 常由多个 CPU 核心共享；
  • 用于减少对 DRAM 的访问。

具体的 Cache 层次、容量和共享方式会因处理器型号而不同。

23. Cache 缺失的主要类型

Cache Miss 通常可以分为三类。

23.1 强制缺失

强制缺失也叫冷缺失。

当某个数据块第一次被访问时，它不可能已经存在于 Cache 中，因此必然发生缺失。

23.2 容量缺失

Cache 容量有限。

如果程序在一段时间内需要访问的数据总量超过 Cache 容量，一部分数据就会被替换。

当 CPU 再次访问被替换的数据时，就会发生容量缺失。

23.3 冲突缺失

在直接映射或组相联 Cache 中，某些主存块可能映射到同一 Cache 位置。

即使 Cache 中还有其他空间，这些数据也可能互相替换，从而产生冲突缺失。

提高 Cache 的相联度，可以减少冲突缺失。

27. 总结

Cache 的核心作用是利用程序的局部性，将 CPU 近期可能访问的数据提前保存在高速存储器中。

CPU 访问一个地址时：

  1. 将地址拆分为 Tag、Index 和 Offset；
  2. 使用 Index 找到对应的 Cache 组；
  3. 比较 Tag，并检查 Valid；
  4. 如果匹配，则 Cache 命中；
  5. 使用 Offset 找到 Cache Line 内的目标数据；
  6. 如果没有匹配，则发生 Cache Miss；
  7. 从下一层 Cache 或主存中读取整条 Cache Line；
  8. 必要时按照替换算法替换旧 Cache Line；
  9. 如果旧 Cache Line 是脏数据，则先写回；
  10. 将新数据装入 Cache，再返回给 CPU。

可以用一句话概括：

CPU 使用 Index 查找位置，使用 Tag 验证数据身份，使用 Offset 定位块内数据；命中时直接读取，缺失时从下一层搬入一整条 Cache Line。
