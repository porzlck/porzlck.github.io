树的Prufer序列与父序列的相互转换

发表于2026-06-25|更新于2026-06-25|算法
|浏览量:

Prufer 序列

Prufer 序列可以将一棵带标号的 nnn 个节点的树，用一个长度为 n−2n - 2n−2，值域为 [1,n][1,n][1,n] 的序列表示。

也就是说，nnn 个点的完全图的生成树与长度为 n−2n - 2n−2、值域为 [1,n][1,n][1,n] 的序列之间存在双射。

Prufer 序列可以方便地解决一类树相关的计数问题。

例如凯莱定理：

n 个点的完全图的生成树数量为 nn−2n \text{ 个点的完全图的生成树数量为 } n^{n - 2} n 个点的完全图的生成树数量为 nn−2

原因是长度为 n−2n - 2n−2 的 Prufer 序列中，每一位都有 nnn 种选择，所以总数为：

nn−2n^{n - 2} nn−2

对树构造 Prufer 序列

Prufer 序列的构造过程如下：

每次选择当前所有叶节点中编号最小的一个，将其删除，并在序列中记录它连接到的那个节点。

重复这个过程 n−2n - 2n−2 次后，树中只剩下两个节点，算法结束。

需要注意的是，这里的叶节点指的是当前度数为 111 的节点。

O(nlogn) 做法

直接按照定义模拟即可。

用小根堆维护当前所有叶节点，每次取出编号最小的叶节点 xxx。

设 xxx 当前连接的节点为 yyy，则将 yyy 加入 Prufer 序列，然后删除 xxx。

如果删除 xxx 后，yyy 的度数变成 111，则将 yyy 加入堆中。

由于每个点最多入堆、出堆一次，所以复杂度为：

O(nlog⁡n)O(n \log n) O(nlogn)

O(n) 做法

可以使用一个指针代替堆来找到当前编号最小的叶节点。

维护一个指针 ppp，始终指向当前可能的最小叶节点。

每次删除 ppp 指向的叶节点后，只会影响它相邻的那个节点，因此最多只会产生一个新的叶节点。

设新产生的叶节点为 xxx。

如果 x<px < px<p，说明它比当前指针还小，那么它一定是当前编号最小的叶节点，应该立即删除它。

如果 x>px > px>p，则不用立刻处理，因为指针 ppp 继续向后移动时一定会扫到它。

因此每个点最多被指针扫到一次，复杂度为：

O(n)O(n) O(n)

Prufer 序列的性质

从 Prufer 序列的构造过程可以得到两个重要性质。

性质 1

构造完 Prufer 序列后，原树中会剩下两个节点，其中一个一定是编号最大的点 nnn。

原因是每次删除的是当前编号最小的叶节点。

如果 nnn 是叶节点，只要当前树中还剩超过两个节点，就一定还存在另一个叶节点，而另一个叶节点的编号一定小于 nnn。

所以 nnn 不会在前 n−2n - 2n−2 次操作中被删除。

性质 2

每个节点在 Prufer 序列中出现的次数等于它在原树中的度数减 111。

即：

cnti=degi−1cnt_i = deg_i - 1 cnti​=degi​−1

也就是：

degi=cnti+1deg_i = cnt_i + 1 degi​=cnti​+1

其中 cnticnt_icnti​ 表示点 iii 在 Prufer 序列中出现的次数。

因此，没有在 Prufer 序列中出现过的点一定是叶节点。

用 Prufer 序列构造树

根据 Prufer 序列的性质，我们可以先得到原树中每个节点的度数。

对于每个节点 iii：

degi=cnti+1deg_i = cnt_i + 1 degi​=cnti​+1

其中 cnticnt_icnti​ 表示 iii 在 Prufer 序列中出现的次数。

然后依次枚举 Prufer 序列中的每个点。

每次选择当前编号最小的度数为 111 的点 xxx，将它与当前枚举到的 Prufer 序列中的点 yyy 连边。

然后同时减少 xxx 和 yyy 的度数。

重复 n−2n - 2n−2 次后，只会剩下两个度数为 111 的点，其中一个一定是 nnn，最后将它们连边即可。

O(nlogn) 做法

用小根堆维护所有当前度数为 111 的点。

每次取出编号最小的点 xxx，将它和当前 Prufer 序列中的点 yyy 连边。

然后执行：

degx←degx−1deg_x \leftarrow deg_x - 1 degx​←degx​−1

degy←degy−1deg_y \leftarrow deg_y - 1 degy​←degy​−1

如果 degy=1deg_y = 1degy​=1，则将 yyy 加入堆中。

复杂度为：

O(nlog⁡n)O(n \log n) O(nlogn)

O(n) 做法

和线性构造 Prufer 序列类似，也可以使用一个指针代替堆。

维护指针 ppp，指向当前编号最小的度数为 111 的点。

每次将 ppp 指向的点 xxx 与当前 Prufer 序列中的点 yyy 连边。

连边后减少两个点的度数。

如果 yyy 的度数变成 111，并且 y<py < py<p，那么 yyy 就是当前编号最小的叶节点，下一次直接使用 yyy。

否则让指针 ppp 继续向后移动，直到找到下一个度数为 111 的点。

由于指针最多扫过每个点一次，所以复杂度为：

O(n)O(n) O(n)

【模板】Prüfer(Prufer) 序列

题目背景

Prüfer 序列又写作 Prufer 序列。

题目描述

请实现 Prüfer 序列和无根树的相互转化。

为方便你实现代码，尽管是无根树，我们在读入时仍将 nnn 设为其根。

对于一棵无根树，设 f1…n−1f_{1\dots n-1}f1…n−1​ 为其父亲序列（fif_ifi​ 表示 iii 在 nnn 为根时的父亲），设 p1…n−2p_{1 \dots n-2}p1…n−2​ 为其 Prüfer 序列。

另外，对于一个长度为 mmm 的序列 a1…ma_{1 \dots m}a1…m​，我们设其权值为 xor⁡i=1mi×ai\operatorname{xor}_{i = 1}^m i \times a_ixori=1m​i×ai​。

输入格式

第一行两个整数 n,mn,mn,m，表示树的点数和转化类型。

若 m=1m = 1m=1，第二行一行 n−1n-1n−1 个整数，表示父亲序列。
若 m=2m = 2m=2，第二行一行 n−2n-2n−2 个整数，表示 Prüfer 序列。

输出格式

若 m=1m = 1m=1，一行一个整数，表示给出的父亲序列对应的 Prüfer 序列的权值。
若 m=2m = 2m=2，一行一个整数，表示给出的 Prüfer 序列对应的父亲序列的权值。
