Yandex.Algorithm 2011 Round 2 C题解 AC自动机优化DP

发表于2026-06-24|更新于2026-06-24|算法
|浏览量:

CF86C Genetic engineering

题目描述

“多维空间现在已经不流行了，反倒是遗传学问题成了潮流。”——物理学家 Woll 如此想着，于是将研究方向转向了生物信息学。在分析测序结果时，他遇到了以下有关 DNA 序列的问题。我们将 DNA 序列简化为只由大写字母 “A”、“C”、“G” 和 “T” 组成的任意字符串（当然，这只是一个简化的解释）。

设 www 为一个较长的 DNA 序列，s1,s2,...,sms_1, s_2, ..., s_ms1​,s2​,...,sm​ 为若干较短 DNA 序列组成的集合。我们说集合能够过滤 www，当且仅当 www 可以被集合中的这些序列完全覆盖。显然，字符串不同位置对应的子串可以重叠，甚至可以相互覆盖。更正式地说：令 ∣w∣|w|∣w∣ 表示 www 的长度，www 的字符从 111 编号到 ∣w∣|w|∣w∣。那么对于 www 的每一个位置 iii，都存在一对下标 l,rl, rl,r（1≤l≤i≤r≤∣w∣1 \leq l \leq i \leq r \leq |w|1≤l≤i≤r≤∣w∣），使得子串 w[l...r]w[l...r]w[l...r] 恰好等于集合中的某一个 s1,s2,...,sms_1, s_2, ..., s_ms1​,s2​,...,sm​。

Woll 想计算：长度为 nnn，且能被给定集合过滤的不同 DNA 序列的数量。但他不会解决这个问题。请你帮他！你的任务是：求出被集合 si{s_i}si​ 过滤的长度为 nnn 的不同 DNA 序列的数量。

由于答案可能很大，请将其模 100000000910000000091000000009 输出。

输入格式

第一行包含两个整数 nnn 和 mmm（1≤n≤1000,1≤m≤101 \leq n \leq 1000, 1 \leq m \leq 101≤n≤1000,1≤m≤10），分别表示字符串的长度和集合中序列的数量。

接下来的 mmm 行，每行一个序列 sis_isi​，表示集合中的 DNA 短序列。每个 sis_isi​ 是长度不超过 101010 的非空字符串，仅包含 “A”、“C”、“G”、“T” 四个大写字母。集合中可能存在相同的字符串。

输出格式

输出一个整数，表示被集合过滤的 DNA 序列的数量，对 100000000910000000091000000009 取模。

这题不是简单判断模式串有没有出现，而是要求：

每个位置都被覆盖。

所以我们需要一边构造字符串，一边判断当前前缀有没有位置还没有被覆盖。

由于模式串数量少，长度也不超过 10，可以用：

AC 自动机 + DP

先把所有模式串插入 AC 自动机。

每个节点维护一个值：

mx[u]

表示：

当前走到节点 u 时，以当前位置结尾的最长模式串长度。

为什么要取 fail 上的最大值？

比如模式串有：

AT

当前匹配到了：

CAT

虽然 CAT 本身不是模式串，但是它的后缀 AT 是模式串。

所以需要：

mx[u] = max(mx[u], mx[fail[u]]);

意思是：

当前节点也要继承 fail 后缀上的模式串信息。

DP 状态

设：

dp[i][u][j]

表示：

已经构造了前 i 个字符，当前在 AC 自动机节点 u，最后还有 j 个位置没有被完全确认覆盖。

也可以理解为：

前 i - j 个位置已经确定被覆盖
最后 j 个位置还在等待后面的模式串覆盖

因为最长模式串长度为 len，所以如果：

j >= len

就说明最前面的未覆盖位置已经不可能再被后面的模式串覆盖了。

所以 j 只需要枚举：

0 ~ len - 1
状态转移

当前状态是：

dp[i][u][j]

现在添加一个字符 c，走到新节点：

v = tr[u][c]

添加一个新字符后，未覆盖长度先变成：

nj = j + 1

如果当前位置结尾有一个模式串，长度为 mx[v]，并且：

mx[v] >= nj

说明这个模式串可以把最后这段还没确认覆盖的位置全部盖住。

于是：

nj = 0;

否则 nj 不变。

如果：

nj < len

说明这个状态还可能合法，可以转移。

初始化和答案

初始状态：

dp[0][0][0] = 1;

表示还没有构造字符，在根节点，没有未覆盖位置。

最后答案是：

sum dp[n][u][0]

因为构造完长度为 n 的字符串后，不能再有未覆盖的位置。
