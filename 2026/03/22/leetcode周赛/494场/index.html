Q3. 最少删除使剩余元素异或和等于 target

题意

给定数组 nums 和整数 target。

你可以删除任意多个元素，要求剩余元素的异或和恰好等于 target。

求最少删除次数。若无法做到，返回 -1。

转化

删除一些元素，等价于保留一个子序列。

如果我们保留的这个子序列异或和为 target，并且长度尽量长，那么删除次数自然就最少。

所以这题等价于：

在所有异或和等于 target 的子序列中，求最长长度。

设这个最长长度为 mx，那么答案就是：

n−mxn - mx n−mx

DP 设计思路

设 dp[x] 表示当前处理到某些元素后：

  • 异或和为 x
  • 最多可以保留多少个元素

初始时：

  • dp[0] = 0，表示一个数都不选
  • 其余状态为 -1

每次加入一个数 x，有两种选择：

  • 不选它，状态不变
  • 选它，那么原来异或和为 i 的状态会转移到 i ^ x

转移方程为：

v[i⊕x]=max⁡(v[i⊕x],dp[i]+1)v[i \oplus x] = \max(v[i \oplus x], dp[i] + 1) v[i⊕x]=max(v[i⊕x],dp[i]+1)

最后：

  • 如果 dp[target] == -1，说明无法构造，返回 -1
  • 否则返回 n - dp[target]

代码

复杂度分析

设状态总数为 m，则：

  • 时间复杂度：O(nm)
  • 空间复杂度：O(m)

Q4. 统计好子数组的数量

题意

给定一个整数数组 nums。

如果一个子数组中所有元素的按位或结果，恰好等于这个子数组中某个出现过的元素，那么称这个子数组为好子数组。

求好子数组的数量。

性质分析

设某个子数组的按位或为 x。

这个子数组是好子数组，当且仅当：

  • 子数组中存在某个元素，值恰好等于 x

所以问题就变成：

统计有多少个子数组，其区间按位或值恰好在区间中出现过。

经典结论

按右端点枚举子数组。

对于所有以 i 为右端点的子数组，它们的按位或结果种类数最多只有 O(30) 个。

原因是按位或只会让二进制位从 0 变成 1，而整数的有效位数有限，所以不同的结果不会太多。

因此可以维护所有“以当前位置结尾”的不同按位或状态。

状态设计

每个状态维护三项：

  • a：这一组子数组的按位或值
  • b：这一组子数组的数量
  • c：这一组中好子数组的数量

我们用两个数组：

  • pre：上一轮状态
  • cur：当前轮状态

先加入只包含 nums[i] 的子数组：

  • 它的按位或值就是 nums[i]
  • 数量为 1
  • 它显然是好子数组，所以好子数组数量也是 1

然后把 pre 中所有状态都接上 `nums[i]``。

设原状态为：

  • 按位或值 a
  • 数量 b
  • 好子数组数 c

那么接上 nums[i] 后，新按位或为：

x=a∣nums[i]x = a \mid nums[i] x=a∣nums[i]

接着分类讨论：

1. x == nums[i]

说明加入的新元素本身就等于整个新区间的按位或值。

于是这一批新子数组全部都是好子数组，新增数量为 b。

2. x == a

说明 nums[i] 没有提供新的二进制位。

那么新区间是否为好子数组，与原状态完全一致，所以继承 c。

3. 其他情况

说明新区间或值变成了一个新的数，并且它既不是新加入的末尾元素，也不是原来的区间或值。

那么这一批新区间都不是好子数组，贡献为 0。

合并状态

不同旧状态在转移后，可能得到相同的新按位或值。

所以需要把相同按位或值的状态合并：

  • 子数组数量相加
  • 好子数组数量相加

最后把当前所有状态中的好子数组数量加到答案里即可。

代码

复杂度分析

由于每个位置结尾的不同按位或值最多只有 O(30) 个，因此：

  • 时间复杂度：O(30n)
  • 空间复杂度：O(30)
