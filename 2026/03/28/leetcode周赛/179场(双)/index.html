Q2. 恰好看到 K 个人的方向选择

pos 左边有 pos 个人，右边有 n - 1 - pos 个人。

对于 pos 来说：

  • 左边的人只有选 L 才能被看到
  • 右边的人只有选 R 才能被看到

所以除了 pos 自己之外，其余 n - 1 个人里，每个人都对应一种被看到的唯一选法，另一种就是看不到。

于是问题就变成：

在这 n - 1 个人里，恰好选出 k 个人被看到，方案数是：

C(n−1,k)C(n - 1, k)C(n−1,k)

而 pos 自己选 L 还是 R 都不影响答案，所以再乘 222。

最终答案：

2⋅C(n−1,k)2 \cdot C(n - 1, k)2⋅C(n−1,k)

Q4. 统计满足数位和数组的非递减数组数目

因为 0≤arr[i]≤50000 \le arr[i] \le 50000≤arr[i]≤5000，所以每个位置的取值范围只有 500150015001 个，可以直接按“值域”做 DP。

先预处理 0 ~ 5000 每个数的数位和，记为 v[x]。

设 dp[x] 表示当前处理到某一位时，最后一个数恰好为 x 的方案数。

对于第一个位置：

  • 如果 v[x] == digitSum[0]，那么可以直接取 x
  • 所以 dp[x] = 1
  • 否则 dp[x] = 0

处理第 i 位时，如果当前值取 x，需要满足：

  • v[x] == digitSum[i]
  • 数组非递减，所以前一个数必须 <= x

因此转移为：

bp[x]=∑y=0xdp[y](v[x]=digitSum[i])bp[x] = \sum_{y=0}^{x} dp[y] \quad (v[x] = digitSum[i])bp[x]=∑y=0x​dp[y](v[x]=digitSum[i])

否则 bp[x] = 0。

这个前缀和可以用数组 pre[x] 优化：

pre[x]=dp[0]+dp[1]+⋯+dp[x]pre[x] = dp[0] + dp[1] + \cdots + dp[x]pre[x]=dp[0]+dp[1]+⋯+dp[x]

这样就能做到 O(1)O(1)O(1) 转移。

最后把所有 dp[x] 求和即可。
