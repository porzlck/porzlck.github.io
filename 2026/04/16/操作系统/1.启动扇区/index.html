TinyOS 1.启动扇区

发表于2026-04-16|更新于2026-04-15|操作系统
|浏览量:

在电脑刚通电时，CPU 并不知道操作系统在哪里。
于是主板里有一小段固化程序，叫 BIOS。

BIOS 会去磁盘开头读取一小段数据，也就是第一个扇区。

如果这个扇区最后两个字节是 55 AA，BIOS 就会认为它是可启动的。
然后 BIOS 会把这 512 字节的数据加载到内存地址 0x7c00，再让 CPU 从那里开始执行。

所以第一步的核心就是：

  1. 写一个 512 字节的小程序
  2. 让 BIOS 认为它可以启动
  3. 然后它在屏幕上打印一句话

一些关键名词整理

CPU

真正执行指令的硬件。

内存（RAM）

CPU 运行代码时主要读取的地方。BIOS 会把我们的启动扇区从磁盘搬到内存。

磁盘

保存数据的地方。我们的 boot.bin 就是一个可以被当作“启动盘开头”的二进制文件。

扇区（sector）

磁盘读写的基本小块。传统启动扇区大小是 512 字节。

启动扇区（boot sector）

磁盘第一个扇区。如果最后两个字节是 55 AA，BIOS 会把它当作可启动代码。

BIOS

开机时最先运行的一小段程序。它负责把启动扇区加载到内存。

0x7c00

BIOS 通常把启动扇区放到这个内存地址。0x 表示十六进制。

汇编（assembly）

非常接近 CPU 的代码。
C / Python 都太高级了，开机第一阶段还没有运行环境，所以先用汇编。

寄存器（register）

CPU 里面的小变量，比如 ax、ds、sp、si。

实模式（real mode）

CPU 刚启动时所处的早期模式。它比较原始，但可以直接使用 BIOS 服务。

中断（interrupt）

可以理解成“向 BIOS / CPU 请求服务”。
这里的 int 0x10 就是请求 BIOS 帮我们打印字符。

源码里写的是 0xaa55，但生成到二进制文件里会变成 55 AA。
原因是 x86 使用 小端序 little-endian。
.word 0xaa55 表示写入一个 16 位数：
0xaa55
这个数由两个字节组成：
AA 55
但是在 x86 里，多字节数字存到内存/文件时，低位字节在前，高位字节在后。
所以：
.word 0xaa55
最终写进 boot.bin 的真实字节顺序是：
55 AA
再看这两行一起：
.org 510
.word 0xaa55
意思是：
.org 510 把当前位置移动到第 510 个字节
.word 0xaa55 从这里写入两个字节：55 AA
因为启动扇区总共 512 字节，编号是：
第 0 字节 到 第 511 字节
所以最后两个字节就是：
第 510 字节：55
第 511 字节：AA
